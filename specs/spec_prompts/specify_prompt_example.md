# Specify Prompt

## User Story: Создание пользователя через API

**Как администратор системы, я хочу иметь возможность создать нового пользователя через API, чтобы добавить нового сотрудника в систему.**

### Acceptance Criteria:
1. **Запрос на создание пользователя** должен принимать следующие параметры:
   - `username` (строка, обязательное поле)
   - `password` (строка, обязательное поле)
   - `email` (строка, обязательное поле, уникальное в системе)
2. **Ответ API** должен возвращать статус 201 и объект пользователя с полями:
   - `id` (целое число)
   - `username` (строка)
   - `email` (строка)
3. В случае ошибки, например, если email уже существует, API должен вернуть статус 400 с сообщением об ошибке.

### Requirements:
- **Бизнес требования**:
  - Эндпоинт должен быть доступен только для администраторов.
  - Данные пользователя должны быть валидированы перед записью в БД.
- **Безопасность**:
  - Пользовательские пароли должны храниться в безопасном виде (например, с хешированием).

### UI:
- Нет UI для этой фичи, так как это API-эндпоинт.
