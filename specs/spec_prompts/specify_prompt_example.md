# Specify Prompt

## User Story: Создание пользователя через API

**Как пользователь системы, я хочу иметь возможность создать нового пользователя через API, чтобы добавить нового сотрудника в систему.**

### Acceptance Criteria:
1. **Запрос на создание пользователя** должен принимать следующие параметры:
   - `username` (строка, обязательное поле)
   - `password` (строка, обязательное поле)
   - `email` (строка, обязательное поле, уникальное в системе)
2. **Ответ API** должен возвращать статус 201 и объект пользователя с полями:
   - `id` (целое число)
   - `username` (строка)
   - `email` (строка)
3. В случае ошибки, например, если email уже существует, API должен вернуть статус 409 с сообщением об ошибке.

### Requirements:
- **Бизнес требования**:
  - Данные пользователя должны быть валидированы перед записью в БД.
- **Безопасность**:
  - Пользовательские пароли должны храниться в безопасном виде с хешированием.

### UI:
- Без UI для фичи, это API-эндпоинт.
