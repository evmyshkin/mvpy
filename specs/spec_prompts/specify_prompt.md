## User Story 1: Структура ролевой модели

**Как администратор системы**, я хочу иметь разделение полномочий в системе по ролям, чтобы управлять возможностями разных групп пользователей**

### Acceptance Criteria:
1. **В системе есть 3 роли** пользователь, менеджер, администратор.
2. **Никакой дополнительной логики**: достаточно лишь создать роли.

## User Story 3: Роли по-умолчанию

**Как пользователь, я хочу чтобы при регистрации в системе мне присваивалась автоматически роль «пользователь», чтобы начать пользоваться системой.**

### Acceptance Criteria:

1. **Роль по-умолчанию**: при создании пользователя присваивать ему роль «пользователь».


## User Story 4: Проверка роли пользователя

**Как администратор системы, я хочу чтобы проверяла роль пользователя при каждом взаимодействии с системой, чтобы иметь представление о том, какими полномочиями обладает пользователь.**

### Acceptance Criteria:

1. **Система проверяет роль** при каждом запросе пользователя, но пока не реализует никакую логику.
