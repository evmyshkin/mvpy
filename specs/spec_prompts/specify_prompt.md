# Specify Prompt

## User Story: Поиск пользователей через API

**Как пользователь системы, я хочу иметь возможность найти нужного мне пользователя, чтобы получить информацию о нем.**

### Acceptance Criteria:
1. **Запрос на поиск пользователя** находить пользователя по `email`.
2. **Ответ API** должен содержать статус код 200 и информацию о пользователе, кроме его пароля.
3. **Поиск всех пользователей** должен выполняться, если не указан параметр `email`
4. **В случае ошибки**, например, если `email` не найден, API должен вернуть соответствующий статус-код с сообщением об ошибке.

### UI:
- Без UI для фичи, это API-эндпоинт.
