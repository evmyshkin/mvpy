# 005-authenticate-user

## User Story 1: Аутентифиация

**Как пользователь системы, я хочу иметь возможность аутентифицироваться с помощью моего email и password, чтобы пользоваться системой.**

### Acceptance Criteria:
1. **Запрос на аутентификацию** авторизует пользователя по совпадению `email` и `password` с теми, что хранятся в БД.
2. **Ответ API** должен подтверждать успешную аутентификацию и выдавать JWT-токен.
3. **В случае ошибки**: если `email` не найден, пароль некорректен или пользователь неактивен, API должен вернуть соответствующий статус-код и ошибку.

### UI:
- Без UI для фичи, это API-эндпоинт.

## User Story 2: Выход из системы

**Как пользователь системы, я хочу иметь возможность разлогиниться из системы, чтобы прервать сессию использования системы.**

### Acceptance Criteria:
1. **Запрос на выход из системы** сбрасывает сессию пользователя.
2. **Ответ API** должен подтверждать успешный выход из системы.
3. **В случае ошибки**: API должен вернуть соответствующий статус-код и ошибку.

### UI:
- Без UI для фичи, это API-эндпоинт.

## User Story 3: Базовая авторизация

**Как авторизованный пользователь системы, я хочу иметь возможность пользоваться инструментами системы, чтобы иметь все операции над пользователями.**

### Acceptance Criteria:
1. **Авторизованный пользователь** может управлять всеми инструментами API системы.
2. **Неавторизованный пользователь** может вызывать управлять только регистрацией и аутентификацией.
3. **Деактивированный во время активной сессии пользователь** теряет возможность управлять системой, кроме регистрации и аутентификации.

### UI:
- Без UI для фичи, это API-эндпоинт.
